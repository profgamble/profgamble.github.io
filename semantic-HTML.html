Creating Semantic HTML

In this exercise, you will learn how to create semantic HTML for a book chapter. This will include adding a navigation bar, a footer, a main, and an aside.

What is Semantic HTML?

Semantic HTML is HTML markup that clearly describes the meaning of the content it contains, rather than formatting how it appears. This markup makes browsers and developers better able to understand how a webpage is structured and can aid in accessibility and discoverability.

Exercise

In this exercise, you will work with the first chapter of Winnie-the-Pooh as transcribed by Project Gutenberg in an exercise created by me.

1. Download the raw .html file. Load it in your code editor. I am using Dreamweaver in class.

2. Make the Chapter I into a top-level heading (h1) and the subheading into a second-level heading (h2) Nest these in a heading group.

3. Make each image a figure. Do not add a figure caption. Do not resize the images.

4. Make the entire page a main element.

5. Choose three short statements that you'd like to highlight. Create an aside for each of these. You will not be able to see them visualized until after you apply CSS, which we will do in the next exercise.

6. Add a navigation section at the top of the page before your main element. Create three links on it:

  • Main Page (going to https://www.gutenberg.org/ebooks/67098)
  • Next Chapter (going to https://www.gutenberg.org/cache/epub/67098/pg67098-images.html#CHAPTER_II)
  • Chapter Three (going to https://www.gutenberg.org/cache/epub/67098/pg67098-images.html#CHAPTER_III)
